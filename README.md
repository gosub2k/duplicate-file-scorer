# duplicate-file-scorer

The tool reads groups of duplicate files from stdin and generates a bash script that intelligently chooses which file to keep and which to delete.

## Example

```bash
# Mark files in /important as golden, /tmp as trash
cat duplicate_files.txt | stack exec duplicate-file-scorer-exe --golden "/important" --trash "/tmp"
### Command-Line Options

- `-g, --golden DIR` - Mark files in this directory as "golden" (preferred to keep). Can be specified multiple times.
- `-t, --trash DIR` - Mark files in this directory as "trash" (preferred to remove). Can be specified multiple times.
- `-h, --help` - Show help message
```

## Input Format

The tool expects groups of duplicate files on stdin, with each group separated by a blank line:

```
/path/to/file1.txt
/path/to/duplicate1.txt

/path/to/file2.jpg
/path/to/duplicate2.jpg
/path/to/duplicate3.jpg
```

## Output

The tool outputs a bash script with comments showing:
- Which file to KEEP and why
- Which files to REMOVE with the rm commands

Example output:
```bash
#!/bin/bash
# Generated by duplicate-file-scorer

# KEEP:
#  /important/file.txt
#  (golden directory match: /important)
# REMOVE:
rm -v /tmp/file.txt  #trash directory match: /tmp
```

## Scoring Logic

Files are scored based on:
1. Golden directory matches (highest priority)
2. Trash directory matches (lowest priority)
3. Modification time (newer is better)
4. Path depth (shallower is better)
5. Path length (shorter is better)

# Build Steps

## Prerequisites

- [Stack](https://docs.haskellstack.org/en/stable/) - The Haskell Tool Stack

## Building from Scratch

This project uses Stack for build management. The `package.yaml` file contains the project configuration, and Stack will automatically generate the `.cabal` file from it.

1. Clone or download the project:
   ```bash
   cd /path/to/duplicate_file_scorer
   ```

2. Build the project:
   ```bash
   stack build
   ```
   
   Wait a lot. This will:
   - Download and install the appropriate GHC compiler (if needed)
   - Install all dependencies specified in `package.yaml`
   - Generate the `duplicate-file-scorer.cabal` file automatically
   - Compile the project

3. Run the executable:
   ```bash
   stack exec duplicate-file-scorer-exe
   ```
